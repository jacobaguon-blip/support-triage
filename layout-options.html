Recommended Option A: Tabs Above Phase Bar, Per-Run Progress
Run tabs sit at the very top of the right panel. Each tab owns its own phase progress bar. Switching tabs changes both the progress indicator AND the content below (conversation, summary, etc). The Conversation view is scoped to the selected run.

Strengths
Clear mental model ‚Äî each tab is a complete investigation with its own progress + conversation. Easy to compare runs side by side by switching tabs. Conversation view naturally scoped per-run, no confusion about which messages belong where.
Trade-offs
Phase bar re-renders on tab switch. With many runs (5+), tab bar needs horizontal scroll. Need to store per-run phase state in the database (already implemented).
Alternative Option B: Tabs Replace View Toggle, Inline Progress
Run tabs replace the Conversation/Classic toggle. Each run tab shows a small inline progress indicator (dots instead of full bar). The full phase bar stays fixed and always shows the latest run's progress. Past runs show their content as read-only with a "completed" or "superseded" banner.

Alternative Option C: Dropdown Selector + Persistent Phase Bar
Instead of tabs, a dropdown selector at the top lets you switch runs. The phase bar updates to reflect the selected run. Saves horizontal space when there are many runs.
